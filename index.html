App Automation — Login module

Goal: validate the end-to-end login flow on Android (happy path + negative cases). Design: BDD with Cucumber + Page Object Model. Common waits/actions live in BasePage, locators & actions in LoginPage, behavior in Gherkin (login.feature), and glue/reporting in LoginSteps.

What I implemented

  • Structured POM to isolate locators and actions from test logic.
  • Explicit waits centralized to reduce flakiness.
  • Readable scenarios in Gherkin (+ background to prep the app state).
  • Step logs for clear reporting; ready for CI and device/cloud runs.

Key outcomes

  • Faster maintenance: UI changes contained to one page object.
  • Traceable, human-readable test intent via BDD.
  • Stable runs thanks to consistent waits and reusable helpers.
BasePage.java (shared actions) 

// minimal excerpt
public class BasePage {
              
    protected AppiumDriver driver;
    protected WebDriverWait wait;
    
    public BasePage(AppiumDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, 10);
    }
    protected void click(By locator) {
        wait.until(ExpectedConditions.elementToBeClickable(locator)).click();
    }
    protected void writeText(By locator, String text) {
        wait.until(ExpectedConditions.elementToBeClickable(locator)).sendKeys(text);
    }

Web Automation — Login module

Goal: validate login features on the web portal (prepaid, postpaid, incorrect credentials, logout, recovery, registration, and legal links). Design: Selenium + POM + Cucumber. We centralize waits/actions in BasePage, keep locators & actions in LoginPage, describe behavior in Gherkin (login.feature), and glue with LoginSteps. Includes window handling for Terms/Privacy PDFs.

What I implemented

  • Page Object Model: selectors & actions isolated from test logic.
  • Explicit waits via WebDriverWait to reduce flakiness.
  • Reusable helpers: navigate, click, write, text extraction, URL waits.
  • Window/tab handling for Privacy & Terms PDFs with URL validation.

Key outcomes

  • Stable scenarios across prepaid/postpaid and negative flows.
  • Faster maintenance: a UI change touches only one page object.
  • Clear, traceable intent using Gherkin + readable step names.
BasePage.java (web helpers) 

public class BasePage {
    protected static WebDriver driver;
    protected WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(20));

    static {
        ChromeOptions chromeOptions = new ChromeOptions();
        driver = new ChromeDriver(chromeOptions);
    }

    public BasePage(WebDriver driver) { BasePage.driver = driver; }

    protected WebElement find(String xpath) {
        return wait.until(ExpectedConditions.visibilityOfElementLocated(By.xpath(xpath)));
    }

    public void navigateTo(String url) { driver.get(url); }
    public void clickElement(String xpath) { find(xpath).click(); }
    public void write(String xpath, String text) { find(xpath).clear(); find(xpath).sendKeys(text); }
    public String textFromElement(String xpath) { return find(xpath).getText(); }

    public void waitForUrlToContain(String expected) {
        new WebDriverWait(driver, Duration.ofSeconds(20))
            .until(ExpectedConditions.urlContains(expected));
    }
}

login.feature (BDD) 

Feature: Test All features of the Login
Rule: The user can test all features of login page

Background: I am on my attPage
  Given I am on the Att page

@TestAtt
Scenario: Login con numero Prepago
  When the user enters username and password prepaid Dn
  And click on the login button
  Then the results give me the home page

@TestAtt
Scenario: Login with Incorrect Credentials
  When the user enters username and incorrect password
  And click on the login button
  Then Then the system should display an error message indicating that the credentials are incorrect
LoginSteps.java (glue) 

public class LoginSteps {
    LoginPage att = new LoginPage();

    @Given("I am on the Att page") public void navigateToAtt(){ att.navigateToAtt(); }

    @When("the user enters username and password prepaid Dn")
    public void enterCredentialsPrepaid(){ att.writeDn("6441319213"); att.writePass("Mi@tt123"); }

    @And("click on the login button") public void clickLogin(){ att.clickLogin(); }

    @Then("the results give me the home page")
    public void validateHome(){ Assert.assertEquals("Tu saldo es:", att.resultLogin()); }

    @When("the user enters username and incorrect password")
    public void incorrect(){ att.writeDn("6441319213"); att.writePass("mi@tt123"); }

    @Then("Then the system should display an error message indicating that the credentials are incorrect")
    public void shouldSeeError(){ Assert.assertTrue("Contraseña incorrecta", true); }
}
